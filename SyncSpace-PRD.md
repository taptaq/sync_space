## 01 产品愿景

面向 ASD、ADHD、HSP 等神经多样性人群，做"每天真的会打开"的内在状态预警与自我协议工具。

> 神经多样性人士最大的痛点之一，不是"情绪不好"，而是总是事后才知道自己过载了。

SyncSpace 不做通用情绪管理——那条赛道已经有 Calm、Woebot、Wysa 几百个产品。SyncSpace 只做一件事：**补上神经多样性人士缺失的那 30 分钟预警窗口**。

别人好像有内置的预警系统，在过载前就知道"我快不行了，需要撤退"。而 ASD 人士的 interoception（内感受）困难让他们总是在 meltdown 之后才意识到自己过载了。ADHD 人士对自己的精力曲线没有稳定感知。HSP 会把别人的情绪当成自己的，分不清哪些是吸收来的。

SyncSpace 通过被动信号推断用户的内在状态，在临界点前 30 分钟触发用户自己预设的协议——"你和自己约定过：气压升高时先撤退 15 分钟。现在去吗？"一次 tap，协议执行。

**核心定位：** SyncSpace 不是一个 AI 情感助手，是一份会自己提醒你、会观察你、偶尔给你递张便条的"内在说明书"。AI 是秘书，协议才是主角。

**比赛对齐：** 清华大学无障碍发展研究院"AI 无障碍工具"方向 · 精神及智力障碍辅助类别。评分维度：真实、可用、共创、共益。推荐模型：Qwen3.7-Plus / Qwen3-VL / Qwen3.5-Omni / Wan2.7。

## 02 目标用户与真实动机

不是"所有想管理情绪的人"，是神经多样性谱系中内感受困难最刚性的人群。

### 用户分层

#### ASD `MVP` — 第一层 · 最刚性需求

- **真实痛点：**"我总是到 meltdown 了才知道自己过载了。别人好像提前就知道，我不知道他们怎么做到的。"
- **产品命中：**被动信号 + 趋势预警，在过载前 30 分钟发出"气压升高"提醒，而非等用户自己发现。
- **卸载触发：**预警不准两次——一次误报"又在吓我"，一次漏报"崩了它也没提醒我"——信任不可逆。

#### ADHD `Roadmap` — 第二层 · 架构已支持

- **真实痛点：**"我有 47 个规划 App，全是前三天用、第四天忘。不是因为不好用，是我根本想不起来打开。"
- **产品命中：**不靠用户主动打开。用被动信号（签到响应延迟、屏幕使用模式）推断状态，在需要时主动推送协议触发。
- **卸载触发：**需要用户每天主动签到——ADHD 的核心困难就是维持日常系统。

#### HSP `Roadmap` — 第二层 · 架构已支持

- **真实痛点：**"聚会回来我会莫名其妙低落两小时，后来才明白我吸收了太多人的情绪，但当时分不清。"
- **产品命中：**"情绪吸收量"轴 + 协议触发："吸收量已满，接下来 1 小时建议不接收他人情绪输入。"
- **卸载触发：**把 HSP 的敏感框架为"问题"而非"特质"——一旦感觉被诊断就会走。

### 人群特化签到维度

这是产品的护城河。市面所有情绪 App 测的是 valence（开心/难过）和 arousal（高唤/低唤）——这是 1980 年代的情绪二维模型。SyncSpace 测的是 sensory load、dopamine battery、emotional absorption——这些维度在消费级产品里几乎没人碰过。

| 人群 | 签到轴 1 | 签到轴 2 | 签到轴 3 | 自我协议样例 |
|---|---|---|---|---|
| **ASD** `MVP` | 感官负载（声/光/触） | 社交电量剩余 | 可预测性需求 | 感官负载>7 时，15 分钟内撤退到安静空间 |
| **ADHD** | 注意力聚焦窗口 | 多巴胺电量（非"精力"） | 任务启动阻力 | 启动阻力高时，先做 2 分钟微任务破冰 |
| **HSP** | 情绪吸收量 | 环境刺激强度 | 边界完整度 | 吸收量满时，接下来 1 小时不接收他人情绪输入 |

**共益叙事：** MVP 服务 ASD，架构已支持 ADHD / HSP 的协议包扩展。同一个 Relationship Protocol 结构，只是协议内容换了。银发族的记忆协议、慢性病患者的自我管理协议、产后母亲的恢复协议，都是"当 X 发生时我给自己约定 Y"。设计给残障，受益所有人。

## 03 设计哲学

三条原则，每一条都直接对应神经多样性人群的真实需求。

### 可预测 > 完美沟通

对于神经多样性人群而言，可预测远比完美更重要。所有交互可预测、可暂停、可退出、可协商。天气卡不是"诊断"，是低干预建议。AI 偶尔说一句让你愣住的话，但绝不替你做决定。

### 被动 > 主动自报

需要这个工具的人恰恰最难准确自评状态。所以产品不靠用户主动签到，靠行为推断：签到通知响应延迟、连续跳过签到、滑块犹豫时长、反思文字的语义密度。用行为推断状态，而非纯靠自报。

### 协议 > AI 建议

所有心理健康 App 的逻辑都是"AI 告诉你该怎么做"。SyncSpace 的逻辑是"你自己决定，AI 帮你记住并在对的时候提醒"。这是从 top-down 到 bottom-up 的范式翻转，对一直被"被告知该怎么生活"的神经多样性人群来说，这个权力反转本身就是尊重。

### 气候类型，非好坏天气

天气是外部的、"坏天气"是负面的。神经多样性权利运动明确反对把神经特质框架为"需要修复的问题"。"你的大脑今天有暴风雨"可能让一些 ASD 自倡导者感到被病理化。

SyncSpace 不用"好天气/坏天气"，用**气候类型**。晴天不是"好"，暴风雨不是"坏"，它们只是不同的气候条件，各有适合的活动。闷热待雨不是坏天气，只是提示你"适合低感官活动，不建议嘈杂环境"。

## 04 产品架构

三层结构 + 一个大脑。补上的触发层是把协议从文档变成活的东西的关键。

```
感知层 · 不靠自报，靠行为推断
├── 被动信号（签到延迟·跳过·犹豫）
├── 主动签到（15秒三轴滑块）
├── 最低门槛标记（一键"今天崩了"）
└── 反思日志（文字/语音）
        │
        ▼
  Qwen3.7-Plus（观察·推理·建议·记忆）
        │
        ├──> 气候层 · 看见状态
        │    ├── 内在天气卡（气候类型·非好坏）
        │    └── 趋势回放（本周曲线·自己看）
        │
        └──> 触发层 · 协议自动激活
             ├── 阈值监测（被动+主动信号）
             └── 协议触发（"现在去吗？"）
                    │
                    ▼
             协议执行（一次 tap 执行或推迟）
                    │
                    └──> 学习反馈 ──> 感知层
```

### 三层职责

#### 感知层 · 不靠自报，靠行为推断

- **被动信号（用户无感知）：**签到通知响应延迟（慢=低能）、连续跳过签到（=已过载）、签到时滑块犹豫时长、过去 N 小时屏幕亮度变化、屏幕使用模式。这些信号不需要用户做任何额外动作。
- **主动签到：**15 秒三轴滑块，人群特化维度（ASD 测感官/社交电量/可预测性，ADHD 测聚焦窗口/多巴胺电量/启动阻力）。用户主观输入，但不是唯一信号源。
- **最低门槛标记：**一键"今天崩了"。当用户处于无法组织语言的状态时，只需一个 tap 捕捉事件。支持语音乱说一通让 AI 整理。复盘可以延迟到第二天。
- **反思日志：**文字或语音，AI 做语义密度分析，从日志中自动提取候选协议规则。

#### 气候层 · 看见状态

- **内在天气卡：**基于感知层信号 + AI 推理生成。显示当前气候类型（如"闷热待雨""晴朗微风"），适合/不建议的活动。气候类型而非好坏——暴风雨不是坏天气，只是不同气候。
- **趋势回放：**本周签到曲线。第一周只做记录回放，让你自己看趋势。第二周开始 AI 才有足够数据做条件关联和模式发现。把"AI 发现"作为惊喜而非承诺。

#### 触发层 · 协议自动激活

- **阈值监测：**AI 持续监测被动+主动信号。当感官负载趋势上升、连续两小时高值、或匹配某条协议的触发条件时，进入触发状态。
- **协议触发：**推送通知——"你的气压在升——你和自己约定过：这时候先撤退 15 分钟到安静空间。现在去吗？"用户一次 tap 执行（开始 15 分钟计时）或推迟（30 分钟后再提醒）。
- **关键设计：**没有触发层，天气卡就是好看的日记；有了它，天气卡变成预警系统。这是整个 V2 最关键的补丁。

#### 我的气候页交互逻辑

- **趋势图：**三轴可切换查看。数据点为每次签到的值，用平滑曲线连接。第一周无 AI 分析注释；第二周起在异常点上方出现观察气泡。
- **AI 观察：**每周日晚上生成，最多 1 条。格式固定。用户三选一：接受（自动生成协议）/ 修改（进入编辑）/ 忽略（不记录，但不影响下次观察）。
- **协议列表：**按最近执行时间排序。每条协议可展开查看执行历史。暂停的协议灰色显示。AI 候选协议以虚线边框区分。
- **协议自动提取：**AI 从反思日志中识别"当 X 时我做了 Y，效果不错"的模式，自动生成候选协议。用户不维护协议表，只需接受/拒绝/修改。

### 页面 3：回看 · 过载后循环

> 5 分钟 · 或延迟到第二天 · 事后记录 + AI 解读 + 沉淀新协议

#### 事后记录

从崩溃标记进入。展示 AI 整理的语音转文字（斜体灰色）。如果用户只标记了时间没有录音，显示"你想补记一下发生了什么吗？"不强制。

#### AI 解读

三段式——事件描述 / 情绪翻译 / 需求识别。

```

│ AI 解读                              │
```

```
14:05  协议触发 · 已执行
       "撤退 15 分钟" · 执行完毕
```

#### 回看页交互逻辑

## 06 神经自我协议系统

产品最值钱的数据结构。从"和伴侣的关系协议"变为"和自己神经特质的协议"，结构一行不改。

> ASD 人士和自己感官敏感度的关系，本质上也是一段"关系"——需要协商、需要边界、需要协议。

### 协议数据结构

```json
{
  "id": "protocol_001",
  "trigger": {
    "type": "threshold",
    "axis": "sensory_load",
    "operator": ">",
    "value": 7,
    "duration_minutes": 30,
    "passive_signals": [
      "checkin_delay_hours > 2",
      "screen_brightness_trend = up"
    ]
  },
  "action": {
    "type": "retreat",
    "description": "15 分钟内撤退到安静空间",
    "duration_minutes": 15,
    "timer": true
  },
  "source": "manual",
  "status": "active",
  "execution_count": 12,
  "last_executed": "2026-07-08T14:05:00Z",
  "created_at": "2026-06-15T10:00:00Z",
  "user_confidence": 0.85
}
```

### 协议来源与生命周期

| 来源 | 说明 |
|---|---|
| **手动创建** | 用户在 Onboarding 或协议管理页主动创建。用户自己定义触发条件和约定动作。 |
| **AI 建议提取** | AI 从反思日志中识别"当 X 时我做了 Y，效果不错"的模式，自动生成候选协议。用户只需接受/拒绝/修改。解决协议疲劳问题。 |
| **崩溃复盘提取** | 从崩溃复盘中提取。AI 分析触发条件和用户事后认为有效的应对方式。这是最有价值的协议来源——从痛苦中长出来的规则。 |

### 协议触发条件类型

| 触发类型 | 说明 | 样例 |
|---|---|---|
| `threshold` | 某轴值超过/低于阈值并持续 N 分钟 | `sensory_load > 7 for 30min` |
| `time` | 特定时间点或时间段 | `every 90min during hyperfocus` |
| `context` | 特定环境上下文（未来 Qwen3.5-Omni） | `environment_noise = high` |
| `behavior` | 被动信号推断的行为模式 | `checkin_skipped >= 2 consecutive` |

> **AI 的角色边界：** AI 永远不会修改协议，只能建议。协议的触发、执行、暂停、删除全部由用户决定。AI 的工作是：记住协议、在合适的时刻提起它、帮用户看见自己注意不到的模式。一旦 AI 开始替用户做决定，产品就死了。

## 07 触发系统详解

这是 V2 最关键的补丁。没有触发层，天气卡就是好看的日记；有了它，天气卡变成预警系统。

### 真实触发链路：一个 ASD 用户的下午

```
14:00  被动信号采集（用户无感知）
       ├ 上午签到通知 3 小时才响应
       ├ 签到时感官滑块犹豫 8 秒
       └ 过去 2 小时屏幕亮度持续调高
            │
14:02  AI 推理（Qwen3.7-Plus）
       ├ 感官负载趋势上升
       ├ 预测 30 分钟后过临界点
       └ 匹配协议"感官负载持续升高"
            │
14:02  天气卡更新
       └ 气压升高中 · 当前气候：闷热待雨
         适合：低感官活动 · 不建议：嘈杂环境、重要决策
            │
14:05  协议触发推送
       └ "你的气压在升——你和自己约定过：
          这时候先撤退15分钟到安静空间。现在去吗？"
            │
14:05  用户点击"去"
       └ 开始 15 分钟计时 · 记录执行
            │
       学习反馈 ──> 感知层
```

### 被动信号采集清单

| 信号 | 采集方式 | 推断含义 | MVP |
|---|---|---|---|
| **签到通知响应延迟** | 通知发送 → 用户点击签到的时间差 | >2h = 低能日；>4h = 可能已过载 | P0 |
| **连续跳过签到** | 连续 N 次签到未响应 | ≥2 次 = 已进入过载/shutdown 状态 | P0 |
| **滑块犹豫时长** | 手指在滑块上停留 → 松开的时间 | >5s = 内感受困难加剧，状态模糊 | P0 |
| **反思文字语义密度** | AI 分析日志文字的情感强度/重复词 | 高重复 + 高情感 = 过载前兆 | P1 |
| **屏幕亮度趋势** | 过去 N 小时亮度变化方向 | 持续调高 = 感官敏感度下降（耗尽信号） | P1 |
| **屏幕使用模式** | App 切换频率/停留时长 | 高频切换 = 注意力漂移（ADHD 相关） | P2 |
| **环境声音/光线** | Qwen3.5-Omni 多模态感知 | 环境过亮/过吵 = 外部感官输入升高 | P2 |

### 触发频率与疲劳控制

- **每日上限：**协议触发推送每天最多 3 次。超过后即使阈值命中也只更新天气卡，不推送。
- **推迟上限：**同一条协议最多推迟 2 次。第 3 次阈值命中时不再推送，避免疲劳。天气卡仍显示预警状态。
- **关怀推送：**连续跳过 2 次签到后，发送一条关怀推送（非催促）："今天还没见到你。没事的话点一下这里就好。"一个 tap 确认安全，不需要签到。
- **静默时段：**用户可设置静默时段（如 22:00-08:00），期间不推送任何触发。天气卡仍更新，用户主动查看时可见。

## 08 AI 模型映射

对齐比赛推荐模型。一个 Agent、一个人格、一个推理模型。

| 功能 | 推荐模型 | 用在哪 | 输入/输出 |
|---|---|---|---|
| **关系推理 / 天气卡生成 / 协议触发判断 / 情绪翻译** | **Qwen3.7-Plus** | 整个大脑。感知层推理 + 气候层生成 + 触发层判断 + 回看页解读 | 入：被动信号+签到数据+协议库+历史日志 / 出：气候类型+活动建议+触发判断+情绪翻译 |
| **签到时的图片/表情理解** | **Qwen3-VL** | 多模态输入。用户可拍照签到（如拍环境光线），AI 识别感官环境 | 入：图片 / 出：环境感官负载评估 |
| **环境感知（声音/光线）** | **Qwen3.5-Omni** | 被动环境信号采集。未来用于 context 类型协议触发 | 入：麦克风/摄像头（用户授权）/ 出：环境噪音/光线等级 |
| **日记配图生成** | **Wan2.7** | 回看页的视觉回忆增强。为复盘记录生成氛围插图 | 入：日志文本摘要 / 出：柔和氛围插图 |

### 四种 Prompt 模式（同一个模型）

原设计的四个独立 Agent 合并为一个模型四种 prompt 模式。更一致、更省、更好调。ASD 用户最需要的是一致性和可预测——四个声音恰恰破坏这一点。

| 模式 | 输入 | 输出 |
|---|---|---|
| **观察** | 本周所有签到+被动信号+执行记录 | 一条模式观察。"你过去 N 次 [事件] 前的 M 分钟，都 [行为]。"每周日晚上生成。 |
| **翻译** | 用户反思日志原始文本 | 三段式解读——事件描述/情绪翻译/需求识别。 |
| **建议** | 观察结果 或 崩溃复盘 | 协议候选草案。"当 X 发生时，给自己 Y。" |
| **记忆** | 全部历史数据 | 天气卡生成+触发判断+协议置信度更新。持续学习。 |

## 09 设计风格指南

贴合神经多样性人群的感官特质。低刺激、高可预测、温柔不侵入。

### 核心原则：低感官刺激

ASD 用户的感官敏感度远高于神经典型者。一个对 NT 来说"还行"的界面，对 ASD 用户可能就是过载源。设计的第一原则不是"好看"，是**不造成额外的感官负担**。

### 色彩

| 用途 | 色值 | 说明 |
|---|---|---|
| **底色** | `#FAF7F2` 暖白/奶油色 | 不用纯白——纯白对感官敏感者过于刺眼 |
| **主色** | `#6B5FA0` 柔和雾紫 | 不用高饱和紫——低饱和度降低视觉刺激 |
| **辅色 1** | `#6B9E8A` 鼠尾草绿 | 自然色调带来安全感 |
| **辅色 2** | `#C4956A` 暖陶色 | 温暖、不刺激 |
| **正文色** | `#3A352F` 暖深灰 | 不用纯黑（#000）——纯黑对比度过高 |
| **次要文字** | `#8A8074` 暖灰 | 低对比但可读 |
| **警告色** | `#C4715A` 降饱和陶土红 | 不用高饱和红/橙 |
| **边框** | `#E0D9CC` 暖米色 | 柔和分隔 |

### 字体

- **标题：** Instrument Serif——柔和衬线体，圆润不尖锐，带有温暖感
- **正文：** Outfit——圆润无衬线体，字母间距舒适，x-height 适中不拥挤
- **等宽：** JetBrains Mono——用于数据值、协议代码
- **字号：** 正文 15px 最小 14px，不追求信息密度
- **行高：** 1.8，给文字呼吸空间
- **禁忌：** 不用细体（font-weight < 400）。不用紧凑字母间距——紧凑感造成视觉焦虑

### 间距与布局

- **大间距：** Section 间距 64px，Card 间距 20px。比常规 App 更宽松——空间本身就是降低焦虑的元素
- **圆角：** 卡片 12px，大容器 20px，按钮/标签 999px（全圆角）。圆润 = 安全感，尖锐 = 威胁感
- **禁忌：** 不用硬阴影（blur < 8px）。用柔和大范围阴影（0 2px 12px rgba(...)），模拟自然光下的柔和层次

### 动效

- **过渡：** 所有过渡 200-300ms ease-out。不用 ease-in-out（回弹感不适）。不用 spring 物理动画（不可预测）
- **方向一致性：** 左箭头 = 内容从右往左滑入，右箭头 = 从左往右滑入。不可混用
- **禁忌：** 不用闪烁动画。不用突然出现的弹窗（用 slide-up + backdrop fade）。不用视差滚动。加载动画用缓慢进度条，不用旋转 spinner

### 天气卡气候类型映射

| 气候类型 | 含义 | 视觉表现 |
|---|---|---|
| 闷热待雨 | 感官气压升高 | 柔和紫渐变背景 |
| 晴朗微风 | 状态稳定舒适 | 鼠尾草绿渐变背景 |
| 暖雾弥漫 | 执行功能模糊 | 暖陶色渐变背景 |
| 雷暴预警 | 临界点接近 | 陶土红渐变背景 |

每种气候用柔和渐变背景 + 简洁符号 + 一句话描述。不用真实天气照片——抽象化降低感官负载。符号统一用柔和线条，不用 emoji（emoji 色彩过于丰富）。

### 可预测性设计清单

- **每个页面结构固定：**底部三 Tab 不变。页面内模块位置不随内容变化。
- **推送可预测：**推送时间、频率、格式都可预测。用户知道每天最多 3 次触发推送、3 次签到通知、1 次周日观察。不超出预期。
- **所有操作可撤销：**签到后 5 分钟内可修改。协议执行后可标记"没用"。崩溃标记可删除。用户永远有退路。
- **不使用弹窗打断：**除协议触发推送外，不用任何 modal 弹窗。所有信息在页面流内展示。
- **Toast 反馈：**所有保存/操作成功或失败都有轻量 toast 提示（顶部 slide-down，3 秒自动消失），不用 alert。

## 11 数据与隐私

这是关系冲突数据 + ASD 人士的敏感信息。比赛把"伦理提示"单独列为一个 section，评委对这块非常敏感。

### 数据分层

| 数据类型 | 内容 | 谁可见 |
|---|---|---|
| **签到数据** | 三轴值、时间戳、被动信号 | 仅用户自己 + AI 推理用 |
| **自我协议** | 触发条件、约定动作、执行记录 | 仅用户自己 |
| **崩溃标记** | 时间戳、语音/文字记录 | 仅用户自己 + AI 整理用 |
| **复盘记录** | 事后记录、AI 解读 | 仅用户自己 |
| **天气卡快照** | 历史气候类型记录 | 仅用户自己 |

### 伦理声明

- **知情同意：**Onboarding 时明确告知数据采集范围、用途、存储方式。使用通俗语言（非法律条款），确保 ASD 人士能理解。提供"我不理解"按钮重新解释。
- **非诊断声明：**产品明确声明"SyncSpace 不是医疗工具，不做任何诊断。如果你正在经历严重困扰，请联系专业人士。"首页底部常驻此声明。
- **数据可控：**用户可随时导出全部数据（JSON）。可删除单条记录或全部数据。删除后 AI 模型不保留任何痕迹。
- **共创伦理：**参考 UNSW Disability Innovation Institute 的共创研究伦理指南。ASD 人士参与设计访谈时，确保他们真正理解并同意。访谈内容需用户本人确认后才用于方案。
- **非病理化叙事：**产品文案中绝不使用"障碍""缺陷""治疗""康复"等词。使用"特质""协议""协商""维护"。神经多样性是差异，不是疾病。

### 数据模型（PostgreSQL）

```sql
-- 用户
users (
  id UUID PRIMARY KEY,
  neuro_type TEXT DEFAULT 'asd',     -- asd | adhd | hsp | other
  created_at TIMESTAMPTZ,
  checkin_times JSONB,               -- 自定义签到时间
  silent_hours JSONB                 -- 静默时段
)

-- 签到
checkins (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  axis_sensory FLOAT,
  axis_social FLOAT,
  axis_predictability FLOAT,
  hesitation_ms INT,                 -- 滑块犹豫时长（被动信号）
  notification_sent_at TIMESTAMPTZ,  -- 通知发送时间
  checkin_at TIMESTAMPTZ,            -- 实际签到时间
  response_delay_minutes INT,        -- 响应延迟（被动信号）
  weather_snapshot JSONB             -- 当时天气卡快照
)

-- 协议
protocols (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  trigger JSONB,                     -- 触发条件
  action JSONB,                      -- 约定动作
  source TEXT,                       -- manual | ai_suggestion | crash_reflection
  status TEXT DEFAULT 'active',      -- active | paused | archived
  execution_count INT DEFAULT 0,
  last_executed_at TIMESTAMPTZ,
  ai_confidence FLOAT DEFAULT 0.5
)

-- 协议执行记录
protocol_executions (
  id UUID PRIMARY KEY,
  protocol_id UUID REFERENCES protocols(id),
  user_id UUID REFERENCES users(id),
  triggered_at TIMESTAMPTZ,
  executed_at TIMESTAMPTZ,
  action_taken TEXT,                 -- executed | postponed | ignored
  duration_actual_minutes INT
)

-- 崩溃标记
crash_marks (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  marked_at TIMESTAMPTZ,
  voice_url TEXT,                    -- Supabase Storage URL
  ai_transcript TEXT,                -- AI 整理的文字
  ai_interpretation JSONB,           -- 事件/情绪翻译/需求识别
  reviewed BOOLEAN DEFAULT FALSE
)

-- 反思日志
journals (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  created_at TIMESTAMPTZ,
  raw_content TEXT,                  -- 用户原始输入
  content_type TEXT,                 -- text | voice
  ai_interpretation JSONB,
  extracted_protocol_id UUID REFERENCES protocols(id)
)
```

## 12 Roadmap

从 ASD 单人工具到神经多样性关系基础设施。

| 阶段 | 内容 |
|---|---|
| **Phase 1 · 比赛Demo** | ASD 单人模式。三轴签到 + 天气卡 + 协议触发 + 崩溃标记 + 事后复盘。简单条件逻辑 + Qwen3.7-Plus。2-3 个真实 ASD 用户共创访谈。 |
