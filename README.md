# SyncSpace

SyncSpace 是一款面向神经多样性人群的日常状态观察与自我支持工具。

它帮助你记录当下的感官、能量与执行状态，把难以描述的体验转化为更容易理解的“内在气候”，并在需要时提供简短、具体、可以立即执行的支持方案。

SyncSpace 目前重点支持 ASD、ADHD 等神经特质，也适用于暂时不确定自身特质、希望更好理解日常状态的人。

> SyncSpace 不是医疗或诊断工具，不能替代医生、心理咨询师或其他专业支持。

## 核心理念

SyncSpace 围绕一条持续循环的路径设计：

1. **预警**：提前看见身体、感官、注意力和环境正在发出的信号。
2. **理解**：结合真实经历形成属于自己的规则，并允许旧理解不断被修正。
3. **连接**：把理解转化为对自己的照顾、对他人的表达和可执行的支持协议。

连接之后产生的新经历会再次成为预警与理解的依据。这不是一次性测评，而是一套会跟随使用者变化的个人支持系统。

## 你可以用它做什么

- 快速记录当前状态，不需要写长篇日记。
- 看见感官负载、社交电量、注意力或启动阻力的变化。
- 在过载、卡住或难以表达时，直接选择一个具体支持动作。
- 建立自己的“当……就……”支持协议。
- 回看状态变化、协议执行和过载事件，寻找个人规律。
- 让治愈音景陪你完成一次签到、一次协议，或一段安静的时间。
- 把此刻的气候做成一张明信片，自己决定要不要分享、分享到哪里。
- 根据自己的感官需要减少动画、阴影和视觉刺激。

## 整体使用流程

### 1. 选择适合自己的使用方式

首次进入时，可以选择：

- **我自己用**：由使用者自主签到和管理协议。
- **帮孩子用**：由家长根据可观察行为代理记录，并获得相应的支持提示。

还可以选择当前更相关的神经特质。这个选择只用于调整界面语言和支持方式，不代表诊断结果。切换特质时会显示一句情境化提示，例如 ASD 侧重视官预算与可预测性，ADHD 侧重执行功能与多巴胺电量，其他特质则提示 ASD 和 ADHD 的工具都可见、可按需选用。

### 2. 完成一次快速签到

首页签到通过三个维度记录状态。维度会根据所选特质调整，例如：

- **ASD**：感官负载、社交电量、可预测性。
- **ADHD**：注意力聚焦、行动电量、启动阻力。
- **其他模式**：心情、能量、稳定感等通用维度。

ASD、ADHD 模式会先显示可观察信号，例如“声光变刺”“变化不安”“开始不了”或“反复切换”。不想从抽象感受开始时，可以直接使用这些信号完成记录。ASD 用户默认展开早期信号区，帮助绕开述情障碍带来的内感受词汇负担。

签到可以继承上一次的数值，也可以直接选择“和上次差不多”，减少重复操作。

签到完成后会有一段轻量的即时反馈：连续签到里程碑（1、3、7、14、30 天）、阶段对应的温柔文案，以及手写体短句。过载期不庆祝，只是承认“你注意到了”。反馈会在三秒后自动消失，不增加视觉负担。

### 3. 预警：查看自己的“内在气候”

签到后，SyncSpace 会把状态呈现为内在气候，并区分五个阶段：

1. **平稳期**：适合建设日常节奏和支持系统。
2. **累积期**：压力正在增加，适合提前减载。
3. **预警期**：接近临界点，应减少任务和刺激。
4. **过载期**：优先保证安全、退出刺激环境和休息。
5. **恢复期**：降低要求，给身体和注意力恢复时间。

这些阶段不是好坏评价，而是帮助你选择更合适的行动。每个阶段的叙事会按神经特质分化：ASD 路线侧重感官预算与输入管理，ADHD 路线侧重多巴胺电量与执行功能。同一份“过载”，对 ASD 用户会说“感官预算耗尽，停止一切输入”，对 ADHD 用户会说“多巴胺见底了，此刻不做就是最好的做”。

阶段行动卡片会在气候下方给出**只做一步**的建议，并根据当前阶段推荐一段匹配的治愈音景（详见下文“治愈音景”）。推荐只是建议，不会自动播放。

### 4. 理解：更新自己的个人规则

趋势和回看不是为了给状态打分，而是帮助整理三件事：

> 当我出现什么信号，这通常意味着什么，对我真正有帮助的是什么。

个人规则可以随新经历修改，也可以记录“这次同样符合”，逐步区分稳定规律和偶然事件。

新规则采用一次一个问题的三步流程，并允许选择“暂时不确定”，不要求当下把经历完整解释清楚。

### 5. 连接：生成连接卡与支持协议

个人规则可以转化成给自己的提醒，或一段可以复制给家人、伴侣和支持者的连接卡。分享与否始终由使用者决定。

连接卡可以保存“请用文字”“请不要触碰”“一次只说一件事”等稳定沟通偏好。完成连接后只询问一次是否有帮助，并将结果更新回个人规则。

协议采用简单的 WHEN / THEN 结构：

> 当某个信号出现时，我就执行一个提前约定好的动作。

例如：

> 当环境声音开始让我难以思考时，就戴上降噪耳机并离开五分钟。

系统会优先展示与当前阶段更匹配的协议。协议可以立即执行、推迟、暂停或修改，控制权始终属于使用者。执行后会延时询问一次是否有效，结果会回流到协议库，帮助下次更准。

### 6. 回看与调整

“气候”和“回看”页面用于观察：

- 最近状态如何变化。
- 哪些事件经常出现在过载之前。
- 哪些协议真正有帮助。
- 当前需要增加、修改或停用哪些支持方式。

回看的目标不是追责或要求自己表现得更好，而是逐步建立更适合自己的生活环境。

### 7. 累积触发与连续支持

SyncSpace 会留意连续模式，而不只看单次签到：

- **ASD**：连续 3 次感官输入 ≥ 7，会提示感官卸载协议。
- **ADHD**：连续 3 次启动阻力 ≤ 3，会提示执行支持协议。

这避免了“一次没问题就被忽略”的情况，也让支持更贴近真实累积过程。

## 功能模块

### 预警

日常使用的主要页面，包括：

- 当前内在气候和阶段。
- 三轴快速签到。
- 阶段行动衔接（单一建议 + 推荐音景）。
- 当前阶段的协议推荐。
- 过载或崩溃事件标记。
- 今日记录和轻量反馈。
- 签到 3 次后解锁的气候明信片入口。

### ASD 即时减载

ASD 模式会优先提供不依赖复杂语言的支持入口：

- **刺激太多**：先减少声音、光线或人群中的一个刺激。
- **现在不想说话**：允许暂时不解释，并提供明确的沟通表达。
- **变化让我不安**：只确认接下来的一步、地点和参与者。
- **简化页面**：快速降低动画、饱和度和视觉装饰。

ASD 的早期信号默认展开，覆盖感官敏感、stimming 加剧、社交退缩、shutdown 等可观察线索。系统不会根据表情、声音或“看起来是否平静”推断真实状态。

### ADHD 启动支持

ADHD 模式把零散想法、提醒和任务统一放进一个外部收件箱，避免在多个工具之间寻找：

- 首页随时快速收下，不要求立即分类或完成。
- 理解页一次只整理一条，可选择“作为下一步”“变成理解”或“已处理”。
- 执行时只显示一个下一步，其他内容继续留在收件箱。
- 可主动开启十分钟视觉倒计时；不开启也不影响使用。
- 内容保存在本机，减少依赖工作记忆。

ADHD 的早期信号覆盖注意力漂移、坐立不安、任务频繁切换、冲动话语、启动阻力高等线索。过载期有专属的三条协议：**多巴胺紧急制动**（十分钟发呆）、**身体重置**（高强度短运动）、**大脑外化**（把脑子里所有事项倒在纸上）。

这里没有连续签到、失败惩罚、强制倒计时或强制完成机制。

### 理解

汇集个人规则、状态趋势、阶段变化和事件回看。用户可以新增、修正、验证或移除旧理解。

ADHD 模式会提供可直接选择的理解方向，例如时间没有外显、启动步骤过大、工作记忆已满、精力与任务不匹配或缺少固定启动线索。它们只是待验证的假设，不是人格评价或诊断标签。每条理解只选择一个最容易尝试的支持方式。

应用也会把“我懒、我很差、我又失败了”这类自我定罪，改写成可验证的温和假设，例如启动信号没有外显、第一步仍然太大或当前资源不足。这种重译用于减少羞耻并恢复行动空间，不是否认后果、强行积极，也不替代对身体疼痛、饮食、睡眠或其他健康问题的专业处理。

### 连接

把个人规则整理成连接自己的提醒或连接他人的沟通卡，并提供协议与事件回看的行动入口。

ADHD 模式可以生成短消息、语音提纲或完整说明，并一次只提出一个请求，例如陪自己开始第一步、只提醒一次、确认截止时间或暂时不要催促。回复不要求完整或完美；目标是让信任的人理解节奏并参与解决问题，而不是监督完成。

### 协议

管理个人支持方案，包括：

- 从模板库选择常见方案（按神经特质过滤，按当前阶段排序）。
- 创建自己的触发条件和行动。
- 设置适用阶段与计时长度。
- 暂停、恢复或删除不再适用的协议。
- 执行后延时收集“有帮助 / 一般 / 没帮助”反馈，回流到协议效果跟踪。

### 治愈音景

一组在浏览器里实时合成的环境声音，陪伴签到、协议执行或安静时间。所有声音都是用 Web Audio API 数学合成的，**没有任何音频文件、没有任何网络请求、没有任何版权风险**。

可选的 8 种音景：

- 棕噪音、粉噪音、白噪音（不同频段的柔和屏蔽）
- 雨声（棕噪音底 + 随机水滴振荡器）
- 海浪（棕噪音 + 慢速振幅调制）
- 篝火（棕噪音 + 随机爆裂声）
- Lo-fi（72 BPM 轻节拍 + Am-F-C-G 和弦循环）
- 静音（推荐给 ADHD 过载期）

音景设计遵循神经多样性友好原则：

- 不会自动播放，必须主动点击。
- 切换时 0.5 秒渐入渐出，不会突然刺耳。
- 低感官模式开启时自动暂停，并提示原因。
- 音量和类型会被记住，下次恢复。
- 阶段行动卡片会按当前阶段和神经特质推荐一段匹配的音景（例如 ASD 过载期推荐棕噪音，ADHD 过载期推荐静音），只作为建议。

### 气候明信片

签到满 3 次后，可以在首页看到一张明信片入口。明信片把当前的气候、阶段和一份隐私保护后的气候指纹做成一张可导出的图片。

设计原则：

- **无社交压力**：不会主动推到任何社区，由你自己决定分享到哪里。
- **感官安全**：柔和颜色，静态图片，无闪烁动画。
- **隐私保护**：明信片上只有阶段、主摆幅范围、敏感通道、节奏特征等模式摘要，不含任何原始签到数值。
- **可预测**：导出前实时预览，所见即所得。
- **解锁门槛**：需要 3 次签到才能生成第一张，避免数据过少时给出误导性总结。

明信片用 Canvas 实时绘制，导出为 PNG，全程零依赖、零网络请求。

### 回看

整理签到、协议执行和过载事件。对于创伤或强烈情绪相关内容，SyncSpace 优先记录客观线索，不强迫重复描述痛苦细节。

### 家长代理模式

家长可以根据可观察行为完成签到，并查看更适合当前阶段的支持建议。家长引导卡片按当前阶段给出“可以这样做 / 可以这样说 / 不要做 / 环境调整”四类建议，预警和过载期默认展开“不要做”。该模式用于帮助理解和调整环境，不用于替孩子下诊断或解释其主观感受。

### 可选的智能能力

配置安全的服务端 AI 代理后，可以使用：

- 语音签到与文字整理。
- 过载事件语音补记。
- 基于当前阶段的智能建议。
- 文字转语音。

未配置服务端代理时，应用会自动使用演示数据，不影响其他核心功能。

## ASD、ADHD 与重叠特质

ASD 和 ADHD 使用同一套记录、阶段和协议基础，但在交互层做了深度分化，而不只是内容隔离：

- **阶段叙事**：ASD 侧重感官预算与输入管理，ADHD 侧重多巴胺电量与执行功能。同一阶段对两种特质的描述和措辞都不同。
- **触发逻辑**：ASD 关注连续感官超载，ADHD 关注连续启动阻力。
- **气候描述**：同一气候类型对 ASD 和 ADHD 给出不同的适合 / 不适合清单。
- **早期信号**：ASD 覆盖感官敏感、stimming、社交退缩、shutdown；ADHD 覆盖注意力漂移、坐立不安、任务切换、冲动话语、启动阻力。
- **工具优先级**：ASD 优先感官工具，ADHD 优先执行功能工具。
- **过载期协议**：ADHD 有专属的三条协议（多巴胺紧急制动、身体重置、大脑外化）。

两者可能同时存在。SyncSpace 不认为用户必须被固定在单一标签中。“其他”特质的用户可以同时看到 ASD 和 ADHD 的工具，按需选用，切换时会显示一句情境化提示。

## 隐私与安全

- 签到、协议和回看数据默认保存在当前浏览器本地。
- 治愈音景完全在浏览器内合成，**零音频文件、零网络请求、零版权风险**。
- 气候明信片用 Canvas 本地绘制并导出，不会自动上传任何图片。
- 气候指纹只暴露模式摘要，不暴露任何原始签到数值。
- 应用不进行人脸表情识别、声纹识别或语音情绪识别。
- 第三方模型的 API Key 不应放入任何 `VITE_` 环境变量或浏览器代码中。
- 如需启用真实 AI 能力，应由自己的服务端代理安全保存供应商密钥。

如果你或身边的人正面临自伤、伤人或其他紧急危险，请优先联系当地急救、危机干预服务或可信任的专业人士。

## 本地运行

需要 Node.js 18 或更高版本。

```bash
npm install
npm run dev
```

常用检查命令：

```bash
npm run check
npm run lint
npm run build
```

如需接入自己的服务端 AI 代理，可参考 `.env.example` 配置：

```bash
VITE_QWEN_PROXY_URL="https://your-server.example.com/qwen"
```

供应商 API Key 必须保存在服务端，不能写入前端 `.env`。
