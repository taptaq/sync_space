# SyncSpace

SyncSpace 是一款面向神经多样性人群的日常状态观察与自我支持工具。

它帮助你记录当下的感官、能量与执行状态，把难以描述的体验转化为更容易理解的“内在气候”，并在需要时提供简短、具体、可以立即执行的支持方案。

SyncSpace 目前重点支持 ASD、ADHD 等神经特质，也适用于暂时不确定自身特质、希望更好理解日常状态的人。

> SyncSpace 不是医疗或诊断工具，不能替代医生、心理咨询师或其他专业支持。

## 最近一次可用性优化

- 首次引导只解释“选一个当前困难 → 试一个小办法 → 反馈是否适合”，不再在进入应用前要求用户完成神经特质自评或 ADHD 子类型选择。
- ASD / ADHD 选择只调整内容顺序，并明确声明不作筛查或诊断；自我观察问卷结果仅供与专业人员沟通参考。
- 主路径用“现在 / 我的方法 / 表达需要”替换“预警 / 理解 / 连接”，并把“规则、信号、证据、执行”等词改成生活化表达。
- 每个行动按钮明确说明点开后会发生什么；选择困难后，行动会自动进入视口。
- 新增可选“阅读减负”，增大正文和按钮文字、增加行距；不默认使用可能造成视觉干扰的半词加粗。
- 支持两种并行使用方式：纯软件保持完整功能；“软件 + 手环”只额外提供振动、快捷标记和待用户确认的相对变化线索。
- 表达需要时保留“我的原话 / 真实情况”和“我希望对方怎么做”，草稿只拼接用户内容；原话改变后必须更新并逐字确认，应用不会推测情绪、添加严重程度或自动发送。

## 可选设备接口

`src/lib/wearableBridge.ts` 定义了可插拔 `WearableAdapter`。Web/PWA 默认使用 software-only adapter，不扫描或读取任何设备。未来的华为运动健康、HarmonyOS 穿戴应用或原生宿主可以注册 adapter，提供：

- 授权连接和断开；
- 读取心率/动作相对个人基线的变化摘要；
- 接收手环快捷标记；
- 发送轻微振动提醒。

设备信号进入 App 后只显示“这和你现在有关吗？”，必须由用户确认。系统保存确认后的摘要和误判反馈，不把原始生理数据自动写成个人方法，也不直接判断过载、走神或情绪。

## 核心理念

SyncSpace 围绕一条因果链设计：

```
预警 → 触发干预 → 产生效果数据 → 形成理解 → 支撑连接
```

重心前移到「预警触发干预」。App 只回答三个问题：

1. **现在怎样** — 状态识别 + 困难类型
2. **只做哪一件事** — 默认只显示 1 个可立即执行的动作
3. **有没有用** — 效果反馈回流，按有效性排序规则

连接之后产生的新经历会再次成为预警与理解的依据。这不是一次性测评，而是一套会跟随使用者变化的个人支持系统。

## 你可以用它做什么

- 卡住时打开 App，选一个此刻最大的困难，立刻得到 1-2 个可以做的事。
- 过载时自动进入低感官模式，只给你最低摩擦的一个动作。
- 看见状态小精灵用不同姿态陪伴你（趴着休息 / 注意到什么 / 准备好 / 蜷起来 / 慢慢探头）。
- 建立自己的支持规则，按实际效果排序，无效规则自动下沉。
- 根据自己的感官需要减少动画、阴影和视觉刺激。
- 把 App 装到主屏幕（PWA），每日早/午/晚 3 个时点收到锚点提醒，不用自己记着打开。
- ASD 用户看见"今日感官预算"电池和"明日预告"，把数字变成直觉、把不确定变成可预期。
- ASD 用户在「理解」页记录兴趣沉浸，看清什么能让自己充电。
- ADHD 用户进入专注模式后，每 30 分钟收到一次温和的自我照顾提醒（喝水 / 站起来），不打断只提供出口。

## 整体使用流程

### 1. 选择适合自己的使用方式

首次进入时，可以选择：

- **我自己用**：由使用者自主签到和管理协议。
- **帮孩子用**：由家长根据可观察行为代理记录，并获得相应的支持提示。

还可以选择当前更相关的神经特质。这个选择只用于调整困难类型的默认排序，不代表诊断结果，也不锁死选择。ASD 用户默认看到「感官」排在前面，ADHD 用户默认看到「启动」排在前面，AuDHD / 其他用户不预设。切换特质时会显示一句情境化提示。

### 2. 卡住时打开 App

首页只问一个问题：**你现在卡在哪？**

五个困难类型可选：

| 困难类型 | 干预方向 | 适用场景 |
|---|---|---|
| 感官 | 减法：降低输入 | 声光变刺、人太多、感官过载 |
| 变化 | 加法：增加可预测性 | 计划变了、不确定接下来会发生什么 |
| 启动 | 加法：外部化 + 微启动 | 知道要做但开始不了、卡在第一步 |
| 时间 | 加法：外部化时间结构 | 忘了截止、不知道还剩多久 |
| 交流 | 加法：降低交流摩擦 | 想说但说不出口、需要别人帮忙 |

默认先显示 3 个最相关的困难类型，其他情况按需展开。选定后只显示 1 个首选动作；首选不合适时才展开其他办法。不需要先签到、先建规则或先选轴。

**点击动作不是只弹一个提示**——每个动作都有真正的下一步执行面板：

- 时间可视化 → 输入截止时间，实时计算剩余多久
- 5 分钟微启动 → 内置 5 分钟倒计时器（开始 / 暂停 / 完成），倒计时走完会自动记一条「完成 5 分钟微启动」到收件箱，让自我调节动作进入数据链路
- 纸面外化 / 现在只做 X → 文本输入框，内容自动存入快速收件箱
- body doubling → 提示「打开视频通话，各自做各自的」+ 完成按钮
- 低感官切换 / brown noise / 暂停交流 → 直接执行对应功能

### 3. 自动减载（按神经特质分叉）

产品不要求用户先理解“模式系统”。低感官模式按神经特质差异化触发：

- **ASD**：累积期主动提示「要不要先开低感官？」（拒绝后本次累积期不再弹，离开累积期重置）；过载时自动进入低感官模式，直接给减载动作。
- **ADHD**：不主动提示，过载时也不强制低感官（ADHD 过载路径不同，更需要"紧急制动"而非"降亮度"）。
- **恢复期**（通用）：自动进入恢复模式，只保留一个低要求动作。

用户随时可以退出任何自动减载状态。

### 4. 状态识别与推送分叉

SyncSpace 从签到和崩溃标记识别五个阶段：平稳、累积、预警、过载、恢复。推送策略按神经特质分叉：

- **ASD**：累积期主动推送「要不要先降载」，防止走向过载。
- **ADHD**：不做后台推送，只在用户主动打开 App 时介入。
- **通用**：过载后 24h 内不推送任何签到提醒。

### 5. 签到（折叠态）

签到不再是主路径，降级为首页可展开的折叠入口。默认简单模式：选一个整体感受即可。想精细调整再展开三轴（ASD：感官/社交/可预测性；ADHD：注意力/电量/启动）。语音签到支持最长 20 秒录音。

### 6. 支持规则（合并模型）

个人规则和支持协议合并为统一的「支持规则」：

```
trigger 信号 → action 行动 → effectiveness 效果分
```

- 规则按 effectiveness（helpful 次数 / 总使用次数）降序排列。
- 执行后提供一次可跳过的反馈（有帮助 / 一般 / 没帮助）。
- 无效规则自动下沉，不再占据注意力。
- 没有 streak 连续天数，只有「这条规则过去 30 天用了 X 次，Y% 有用」。

### 7. 连接（差异化版）

连接页针对「自己」和「他人」给出真实不同的功能：

- **连接自己**：3 步内在确认流程——看见信号 → 默念理解 → 给自己 30 秒做一点支持。让规则从"读一遍"变成"真正消化一遍"。信号未出现时也可以提前记住，下次更早识别。
- **连接他人**：生成一条对方视角的短消息（包含 signal + understanding + support，让对方知道"为什么"和"能做什么"），并附带「发送前提醒」卡片（对方可能的第一反应 + 兜底话术），降低沟通焦虑。一键复制到剪贴板。

完成后都会询问一次是否有帮助，反馈回流到规则有效性。分享与否始终由使用者决定。

### 8. 回看与调整

「气候」和「回看」页面用于观察：

- 最近状态如何变化。
- 哪些事件经常出现在过载之前。
- 哪些规则真正有帮助。
- 当前需要增加、修改或停用哪些支持方式。

回看的目标不是追责或要求自己表现得更好，而是逐步建立更适合自己的生活环境。

## 功能模块

### 今日（卡住入口）

日常使用的主要页面，极简设计：

- 状态小精灵用不同姿态陪伴你。
- 「你现在卡在哪？」默认显示三个相关入口，其他情况折叠。
- 每次只展示一个首选动作，其他办法按需展开。
- 过载时自动进入低感官模式，直接给减载包，跳过选择。
- 签到降级为折叠态，不抢主路径。
- 设置入口（齿轮图标）。
- ADHD 用户的快速收件箱。

### ASD 即时减载

ASD 用户在累积期会主动收到「要不要先开低感官？」的提示；过载时自动进入低感官模式：

- 停动效、降亮度、隐藏氛围层、停音景，只留 1 个动作。
- 一键播放 brown noise 感官降载。
- 一键发送「暂停交流」消息。
- 不需要先建规则、先选轴、先签到。

### ADHD 启动支持

ADHD 模式把零散想法、提醒和任务统一放进一个外部收件箱：

- 首页随时快速收下，不要求立即分类或完成。
- 启动困难时选「启动」困难类型，得到 5 分钟微启动 + body doubling + 纸面外化。
- 没有连续签到、失败惩罚、强制倒计时。

### 支持规则（合并模型）

个人规则 + 支持协议合并为统一的「支持规则」：

- 按 effectiveness 降序排列。
- 执行后提供可跳过的轻量反馈。
- 新建/执行/删除，模板库作为起点。
- 无效规则自动下沉。
- **新建规则时支持 AI 参考行动**：填好「信号」后可点击「AI 给参考行动」，基于神经特质 + 信号内容生成 3 条具体行动建议，点一下即填入。未开启 AI 增强或未配置后端时降级为本地模板建议。

### 自定义协议

在「气候」页可以新建自己的协议（当 X 发生时，我给自己约定 Y）：

- 可选轴触发（感官/社交/可预测性 等）或行为触发。
- **THEN 动作区支持 AI 参考建议**：基于触发条件 + 神经特质 + 适用阶段生成 3 条具体动作，每条带推荐时长。点一下即填入动作描述和计时。
- 可标记适用阶段，不标则全阶段通用。
- 协议保存后进入规则池，按效果参与排序。
- **执行后反馈闭环**：协议执行 3 秒后弹出一个轻量 bottom sheet，询问「有帮助 / 一般 / 没帮助」。反馈写入 `executions[].feedback`，回看页趋势图的执行标记会按反馈着色（绿=有帮助 / 橙=未反馈 / 红=没帮助）。
- **推迟重弹**：点击「推迟」后协议不会消失，30 分钟后会自动重新弹出提醒（受每日 3 次上限保护）。应用从后台恢复或重开时会立刻检查是否已到重弹时点。
- **协议聚合效果分**：「理解」页展示每条协议的累计执行次数与有效分（helpful 占比），与支持规则线对称：绿（≥67%）/ 橙（34-66%）/ 红（<34%）。

### 连接（差异化版）

- **连接自己**：3 步内在确认流程（看见 → 理解 → 支持），每步只显示规则的一部分并配引导语，让用户真正消化而非读一遍。信号未出现时可"提前记住"。
- **连接他人**：生成对方视角的短消息（含 signal + understanding + support），并附「发送前提醒」卡片（对方可能反应 + 兜底话术）。
- 一键复制到剪贴板。
- 完成后询问一次是否有帮助，反馈回流到规则有效性。

### 可选音景

全局音景控制器默认显示在屏幕左下角（非 onboarding 页面）。点开可选择 brown noise / white noise / 海浪 等浏览器本地合成的环境声。音景不会自动播放；低感官模式会自动停止声音。它不是核心流程，也不作为治疗效果宣称。

### PWA 与每日锚点提醒

SyncSpace 是一个 PWA：在浏览器打开后通过"添加到主屏幕"即可像原生 App 一样打开（全屏、独立图标、无浏览器栏），不需要应用商店。

设置页可开启「每日锚点提醒」：

- 默认早 / 午 / 晚 3 个时点，每个时点可自定义。
- 到点推送一条系统通知，提醒花 15 秒看看自己。
- 解决 ASD 用户"后知后觉"的核心痛点——不用自己记着打开 App。

通知能力依赖 service worker：App 在前台或后台标签页时所有平台都能收到；App 已关闭时 Android 通常 24 小时内可推，iOS 16.4+ PWA 暂不支持后台推送。真正的"App 关闭也能推"需要后端 VAPID Push 服务（未实现）。

### 感官预算电池（ASD 友好）

首页小精灵下方展示一个电池形状的"今日感官预算"：

- 基于今日签到的感官轴平均值反推剩余预算（0-100%）。
- 三档颜色：绿（≥60% 预算充足）/ 橙（30-60% 在下降）/ 红（<30% 建议降载）。
- 显示趋势（累积中 / 在恢复），帮助 ASD 用户把数字变成直觉。

无今日签到时不显示。

### 明日预告（ASD 友好）

首页展示基于历史签到规律的明日预告：

- 取明日的星期几，在所有历史签到中找同星期几的记录。
- 按小时分桶，找出累积 / 预警 / 过载出现频率最高的时段。
- 至少 3 条同星期几的签到才显示，否则隐藏。
- 文案温柔，不说"风险"，说"那个时段你可以多照顾自己一点"。

满足 ASD 用户对可预测性的核心需求。

### 能量档案（ASD · 理解页）

ASD 用户的特殊兴趣是稳定能量源。理解页底部提供「能量档案」：

- 记录当下兴趣沉浸（输入话题 + 开始/暂停/完成计时）。
- 显示本周充电总时长。
- 按话题聚合，显示 Top 5 充电源排行，看清什么最能让电量回升。

理念：理解自己不只是看什么让你卡住，也包括看清什么能让你恢复能量。

### Hyperfocus 保护（ADHD · 专注模式）

ADHD 的超专注是双刃剑——产出高但容易忘记自我照顾（喝水 / 站起来 / 上厕所 / 吃饭）。专注模式下启用：

- 每 30 分钟弹一个温和提醒，不强制打断（没有刺耳铃声）。
- 不批评（不说"你该休息了"），只提供出口（"喝口水？"按钮 + 继续专注）。
- 关闭后 15 分钟再提醒；重复提醒会逐渐加重语气。

与 5 分钟微启动倒计时分工：微启动解决"启动不了"，Hyperfocus 保护解决"停不下来"。

### 状态小精灵

首页顶部有一只状态小精灵，用不同姿态表示当前阶段（趴着=平稳、坐着留意=累积、警觉=预警、蜷起来=过载、探头=恢复）。小精灵下方有「状态小精灵」说明按钮，点击查看姿态含义。它不催你、不打分，只是陪你看到状态。

### 回看

整理签到、协议执行和过载事件。对于创伤或强烈情绪相关内容，SyncSpace 优先记录客观线索，不强迫重复描述痛苦细节。

### 家长代理模式

家长可以根据可观察行为完成签到，并查看更适合当前阶段的支持建议。家长引导卡片按当前阶段给出“可以这样做 / 可以这样说 / 不要做 / 环境调整”四类建议，预警和过载期默认展开“不要做”。该模式用于帮助理解和调整环境，不用于替孩子下诊断或解释其主观感受。

### 可选的智能能力

配置安全的服务端 AI 代理后，可以使用：

- 语音签到与文字整理。
- 过载事件语音补记。
- 基于当前阶段的智能建议。
- 文字转语音。
- **自定义协议动作 AI 参考建议**：在新建协议时基于触发条件生成 3 条具体动作。
- **支持规则 AI 参考行动**：在新建支持规则时基于信号生成 3 条具体行动。

设置页的 AI 增强区块下会显示生效位置清单，告诉用户开启后具体在 4 个地方生效：

| 位置 | 作用 |
|---|---|
| 语音签到 | 用嘴说代替滑块，15 秒完成 |
| 语音补记崩溃 | 过载后张嘴说就行，不用手打 |
| 协议 / 规则参考动作 | 不知道约定什么动作时给 3 条参考 |
| 气候明信片标语 | 给你的状态生成一句专属话 |

未配置服务端代理时，AI 建议会自动降级为本地模板建议（参考动作仍可用，只是没那么个性化），不影响其他核心功能。

## ASD、ADHD 与重叠特质

ASD 和 ADHD 使用同一套记录、阶段和协议基础，但在交互层做了深度分化，而不只是内容隔离：

- **阶段叙事**：ASD 侧重感官预算与输入管理，ADHD 侧重多巴胺电量与执行功能。同一阶段对两种特质的描述和措辞都不同。
- **触发逻辑**：ASD 关注连续感官超载，ADHD 关注连续启动阻力。
- **气候描述**：同一气候类型对 ASD 和 ADHD 给出不同的适合 / 不适合清单。
- **早期信号**：ASD 覆盖感官敏感、stimming、社交退缩、shutdown；ADHD 覆盖注意力漂移、坐立不安、任务切换、冲动话语、启动阻力。
- **工具优先级**：ASD 优先感官工具，ADHD 优先执行功能工具。
- **过载期协议**：ADHD 有专属的三条协议（多巴胺紧急制动、身体重置、大脑外化）。

两者可能同时存在。SyncSpace 不认为用户必须被固定在单一标签中。“其他”特质的用户可以同时看到 ASD 和 ADHD 的工具，按需选用，切换时会显示一句情境化提示。

## 隐私与安全

- 签到、协议和回看数据默认保存在当前浏览器本地。
- 可选音景完全在浏览器内合成，零音频文件、零网络请求。
- 应用不进行人脸表情识别、声纹识别或语音情绪识别。
- 第三方模型的 API Key 不应放入任何 `VITE_` 环境变量或浏览器代码中。
- 如需启用真实 AI 能力，应由自己的服务端代理安全保存供应商密钥。
- PWA 每日锚点提醒只在本地 service worker 中调度，不涉及任何推送服务器，也不会上传你的签到时间。

如果你或身边的人正面临自伤、伤人或其他紧急危险，请优先联系当地急救、危机干预服务或可信任的专业人士。

## 本地运行

需要 Node.js 18 或更高版本。

### 仅前端

```bash
npm install
npm run dev
```

### 前端 + 后端同时启动

```bash
npm install
npm run dev:all
```

`dev:all` 会用 concurrently 同时启动前端 Vite 开发服务器和后端 Qwen API 代理（Hono）。前端日志显示为蓝色，后端为绿色。

后端代理需要单独安装依赖：

```bash
cd server && npm install
```

并在 `server/.env` 配置 DashScope API Key（参考 `server/.env.example`）。

### 常用检查命令

```bash
npm run check    # TypeScript 类型检查
npm run lint     # ESLint
npm run build    # 构建生产版本
```

### 接入 AI 能力

在 `.env` 配置前端代理地址：

```bash
VITE_QWEN_PROXY_URL="http://localhost:8787"
```

供应商 API Key 必须保存在服务端（`server/.env` 的 `DASHSCOPE_API_KEY`），不能写入前端 `.env`。

未配置代理时，应用会自动使用演示数据，不影响其他核心功能。

### 数据同步

配置 Supabase 后（`.env` 中的 `VITE_SUPABASE_URL` 和 anon key），应用会：

- 启动时匿名登录并拉取云端数据。
- 1.5 秒防抖自动同步到云端。
- 未配置时降级到 localStorage，不影响本地使用。

### 部署到 Cloudflare

前端和后端都适配 Cloudflare：

- 前端：`wrangler.toml` 配置为 assets-only Worker，SPA 回退。
- 后端：`server/wrangler.toml` 配置为 Workers，Hono 框架兼容 Workers 运行时。

```bash
npm run build              # 构建前端
cd server && npm run deploy # 部署后端到 Workers
```


核心链路：预警 → 触发干预 → 产生效果数据 → 形成理解 → 支撑连接
